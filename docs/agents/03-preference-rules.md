# 03 — 偏好规则与罚分

## 设计原则

偏好罚分 >> 单笔订单利润，因此策略第一优先级是**零违规**，第二优先级才是最大化收益。

## 罚分参考（当前评测版本）

| 司机 | 规则 | 单次/每天罚分 | 累计上限 |
|------|------|-------------|---------|
| D006 | 每天至少连续休息 5h | 400/天 | 4000 |
| D007 | 23:00-04:00 不接单不空驶 | 500/天 | 5000 |
| D008 | 每月至少 2 天全休 | 3000/月 | 3000 |
| D009 | 每天 23:00 前回家 + 次日 08:00 前不动 | 600/天 | 6000 |
| D010 规则1 | 每月至少 5 天到达目标点 | 3000/月 | 3000 |
| D010 规则2 | 禁止进入禁入区域 | 3000/次 | 3000 |

## 偏好解析策略

### 运行时解析（非硬编码）
偏好文本由 `get_driver_status()` 返回的 `preferences` 字段提供，
使用 PreferenceParser 通过 LLM 将自然语言解析为结构化 DriverConfig：

```python
# 输入示例
"每天晚上 23 点前必须回到家里（纬度 23.12，经度 113.28），第二天早上 8 点才出门"

# 输出结构
{
    "quiet_window": [23*60, 32*60],  # 跨天
    "must_return_home": True,
    "home_position": {"lat": 23.12, "lng": 113.28},
    "home_return_deadline_hour": 23,
}
```

### 解析失败保护
- 如果 LLM 解析失败，使用保守默认配置
- 保守默认 = 全部品类可接 + 无距离限制 + 每天 8h 休息
- 日志记录解析失败，下次重试

## SchedulePlanner 实现的强制动作

| 条件 | 动作 |
|------|------|
| 当前在禁止活动窗口内 | wait 到窗口结束 |
| 今日连续休息不满要求 | wait 补足差值 |
| 必须回家且来不及（含余量） | reposition 回家 |
| 特殊事件已激活（D010 家事） | 进入事件状态机 |

## 偏好满足验证

每步决策后，StateTracker 维护偏好达成状态：
- `off_days_achieved` — 已完成的全休天数
- `visit_target_days` — 到访目标点的自然日 set
- `longest_rest_today` — 今日最长连续休息
- `quiet_window_violations` — 禁止时段违规计数

月底如果 visit_target_days < 5，主动空驶打卡。
