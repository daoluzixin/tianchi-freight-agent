# 07 - 瓶颈分析 R7 (2026-05-15)

> 历史最佳: 203,876元 | 回退后: 200,254元 | P1优化反降: 190,866元

---

## 一、具体 Bug 级问题

### 1. D009 回家 deadline 跨天计算错误
**位置**: `rule_engine.py` L184-188

`arrive_time` 含 scan_cost 后恰好跨过午夜 → `delivery_day` 被推到次日 → deadline 从"当天23:00"变成"次日23:00" → 本该被拦截的货源被放行。

日志证实 D009 有 4 天未能 23 点前到家，每次罚 900 元 = **3,600 元**。

### 2. D010 家事前瞻触发太晚
**位置**: `schedule_planner` / `model_decision_service`

家事要求 3/10 10:00 到达，Agent 在 3/9 21:16 还接长途单（耗时 24.6h），直接跨过 deadline。SchedulePlanner 没有提前 24-48 小时拦截远途单。

918 分钟 × 5 元 = **4,590 元**。

### 3. D003 货源时效判断缺失
**位置**: `rule_engine.py` L190-200

rule_engine 检查装货窗口时没加 scan_cost（固定 10 分钟偏低），导致多次"空驶到了但货已过期"，累计无效空驶 60km+。总空驶超限 202km 罚 2,000 元；凌晨 2-5 点运输违规 7 次罚 1,400 元 = **3,400 元**。

## 二、参数/阈值级问题

### 4. 安全余量不统一
- `rule_engine`: 45 分钟 buffer
- `cargo_scorer`: 60 分钟 buffer
- `model_decision_service`: 40 分钟 buffer

三处估算彼此矛盾，导致"过滤放行了但评分给了高罚分"或反过来的边缘误判。

### 5. 软约束罚分放大倍率不够
`preference_penalty = base_penalty × 2.5`

对 D009 的 go_home（600 元/天），一单 2000 元毛利仍能覆盖 1500 元罚分，系统仍会接单。但实际接这单会导致当天无法回家。

### 6. off-day 前保护过于保守
off-day 前 12 小时（720 分钟）就停止接单，对短途司机（单程 2-4 小时）浪费半天接单窗口。

---

## 三、模块间信息断裂

### 7. rule_engine 与 scorer 前瞻不一致
两者的时间估算公式和安全余量完全独立，没有共享统一的"时间推演引擎"。一个放行了另一个可能给高惩罚，或反过来。

### 8. supply_predictor 与约束系统断裂
predictor 说"韶关明天有好货值得去"→ position_bonus 鼓励去远处 → 约束系统到了以后才拦截 → 空驶已经产生了。

典型案例：D009 为接熟货去韶关（190km），当天完全无法回家。

### 9. pending_action 信息损失
orchestrator 不回调 `update_after_action` → Agent 用 pseudo_result 猜测结果 → 缺少实际 price/haul_km → 累计收入统计偏低 → daily_review 误判策略太保守 → 放宽 aggression → 更多边缘单 → 更多罚分。

---

## 四、架构级瓶颈

### 10. 决策是"单步贪心"而非"链式规划"

当前架构：每步独立评分 → 选最高分 → 执行。

无法回答：
- "接这单后晚上回得了家吗？"
- "3 天后的家事赶得上吗？"
- "今天的空驶预算还够吗？"

所有"前瞻"都通过在评分公式里加惩罚项来近似，但惩罚项的权重无法精确反映"未来约束被违反的概率"。

**PlanGEN 3.2 尝试做了这件事**，但实现有缺陷（跨天计算错误 + 安全余量不统一），反而加了误拦导致收益下降 13,000 元。

---

## 五、根本性矛盾（最抽象层面）

### 11. "确定性规则" vs "不确定性环境"的张力

系统的所有防护建立在精确的时间/距离估算上：`arrive_time = distance / speed + buffer`

但实际环境充满不确定性：
- scan_cost 变动（5-15 分钟）
- 货源过期
- LLM 超时降级
- 路径估算误差

**加大 buffer = 接单窗口缩小 = 收入下降**

这就是为什么"规则调优到极限了"——用静态 buffer 对冲动态不确定性，收益和安全之间的 **Pareto 前沿已被逼近**。

---

## 改进优先级

| 优先级 | 改进项 | 预期收益 |
|--------|--------|----------|
| P0 | 修复 D009 跨天 deadline 计算 | 减少 ~3,600 罚分 |

## 突破方向

从"静态阈值防护"转向"动态约束感知的链式决策"——让 Agent 在每一步都能看到未来 6-24 小时的约束地图，根据剩余时间预算动态调整接单范围，而不是靠固定的 buffer 值来硬防。

核心实现思路：构建一个统一的 `TimelineProjector`，在接单前快速模拟"接了这单后的状态轨迹"，判断是否会撞到任何硬约束（go_home / quiet_window / family_event / off-day / deadhead_budget）。与 PlanGEN 3.2 的区别在于：共享统一的时间推演逻辑，而非各模块各自估算。
