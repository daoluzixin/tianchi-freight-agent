## 五、测试用例详细说明

### Test 01：司机配置加载完整性

验证 `DRIVER_CONFIGS` 字典包含 D001-D010 全部 10 位司机配置，`get_config()` 正确返回对应 `DriverConfig` 实例。

### Test 02：D001 禁止品类过滤

D001 配置禁止「化工塑料」和「煤炭矿产」。输入 3 条货源（化工塑料/煤炭矿产/食品饮料），验证前两条被过滤，第三条通过。

### Test 03：D001 深圳地理围栏

D001 限定在深圳范围（lat: 22.42-22.89, lng: 113.74-114.66）内作业。输入一条围栏内和一条卸货点在广州的货源，验证围栏外被拒绝。

### Test 04：D003 禁止区域

D003 禁止进入广州白云区（中心 23.30,113.52 半径 20km）。验证卸货点在禁区中心的货源被过滤，远离禁区的通过。

### Test 05：D004 每日最大接单数

D004 每天最多 3 单。模拟 `today_order_count=3` 时无法接单，`today_order_count=2` 时可正常接单。

### Test 06：D005 最大运距限制

D005 限最大运距 100km。构造 146km 和 22km 两条货源，验证超限被滤、合规通过。

### Test 07：D009 特殊货源必接

D009 必须接取 cargo_id=240646 的熟货。即使同时存在 2000 元的高价货源，仍优先选择 600 元的熟货。通过 `ModelDecisionService.decide()` 端到端验证。

### Test 08：D009 夜间回家

D009 每晚 23:00-08:00 必须在家。测试两个场景：距家 40+km 时触发 GO_HOME；已在家时触发 REST。

### Test 09：D001 每日休息约束

D001 每天需连续休息 8 小时。凌晨 3 点（在建议休息窗口 22:00-06:00 内）且未满足休息时返回 REST；休息满足后正常 WORK。

### Test 10：D010 家事状态机

D010 在 sim_minutes=13200（3/10 10:00）触发家事。验证状态从 idle 转为 go_spouse，输出 REPOSITION 到配偶位置。

### Test 11：D003 空驶配额限制

D003 月度空驶上限 100km。当已用 90km 时，空驶 15km 的单（总计 105km）被过滤，空驶 5km 的单（总计 95km）通过。

### Test 12：货源评分排序

验证 CargoScorer 多维评分正确性。高价短途好位置的 HIGH 排第一（794分），高空驶比的 LOW 排最后（239分）。

### Test 13：D001 端到端决策

完整模拟 `decide()` 调用。3 条候选中 C102（化工塑料）被过滤，C101（800元）得分高于 C100（500元），最终选 C101。

### Test 14：安静窗口验证

验证 D005 跨天安静窗口（23:00-06:00）。day1 02:00 返回 REST，10:00 返回 WORK。

### Test 15：软约束品类标记

D010 对「服饰纺织皮革」有软禁止（接了会罚分但不绝对禁止）。验证该品类不被过滤但标记 `is_soft_violated=True`，普通品类无标记。

## 六、发现并修复的问题

### 问题1：`_check_day_rollover` 首次初始化重置计数器

- 现象：D004 在测试中手动设置 `today_order_count=3` 后被 `init_from_status` 清零
- 原因：`_last_day` 初始值 -1，首次调用时 `current_day != _last_day` 为真，触发重置
- 修复：当 `_last_day == -1` 时仅设置 `_last_day = current_day` 并 return，不重置计数器

### 问题2：D009 回家优先级低于安静窗口

- 现象：D009 在 23:30 远离家时返回 REST（原地等待），而非 GO_HOME
- 原因：安静窗口检查（优先级2）排在回家检查（优先级3）之前
- 修复：将 GO_HOME 检查提升到安静窗口之前，确保远离家时先回家
